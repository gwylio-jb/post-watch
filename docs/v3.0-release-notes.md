# Post_Watch v3.0.0 — the evidence release

v2.x made Post_Watch a strong **reference + scanner**: the full ISO/IEC
27001:2022 clause and Annex A control library, a 57-check outside-in
WordPress scanner, gap analysis, a 5×5 risk register, and branded PDF
reports. An ISMS-consultant review of v2.12 concluded it covered roughly
40% of a real engagement — the missing 60% being the evidence layer
certification auditors actually scrutinize.

v3.0 ships that layer. Everything below is local-first, client-scoped,
covered by the versioned migration chain (v6–v9), and exportable.

## The four pillars

**1. Statement of Applicability** (Compliance Hub → SoA)
Live per-client matrix over all 93 Annex A controls: applicability,
justification, implementation status, vault evidence per row. Bulk theme
actions, justification-coverage meter, pre-fill from the latest gap
session (never overwrites curated rows). CSV + branded PDF export.

**2. Findings & CAPA register** (Compliance Hub → CAPA)
Clause 10.2 nonconformity tracking with an enforced lifecycle:
open → action-planned → implemented → verified → closed. No plan without
an owner + due date; no verify without a passed effectiveness check.
Intake hooks on gap items, WP scan findings, internal-audit checklist
lines, and incidents. Overdue actions surface in the Alert Center.
CSV + PDF register exports.

**3. Internal audits + management reviews** (Compliance Hub → two tabs)
Clause 9.2: plan audits with scope presets, execute as a checklist,
raise findings inline, and watch the rolling-12-month ISMS coverage
meter. Clause 9.3: management review records with the standard's fixed
agenda, an auto-frozen posture snapshot at creation, decision tracking,
and minutes PDF export.

**4. KPIs + registers** (Compliance Hub → two tabs)
Clause 9.1 measurement: auto-computed live metrics (compliance %, mean
WP score, open/overdue CAPAs) plus user-defined KPIs with period entries,
targets and trend sparklines. Three registers replace the last external
spreadsheets: training log (with HR-CSV import), incident log (with a
raise-CAPA hook), and an asset register with C-I-A criticality.

## Foundation work shipped along the way

- **v2.13.0 (stability)**: the Tauri sync-dialog bug class closed for
  good (confirm/alert/prompt all async now — backup passphrase entry was
  silently broken in production), a crypto recovery-flow fix that could
  have locked users out after using a recovery code, honest partial-
  import reporting, and a test-runner fix for Node 25.8+.
- **v2.14.0 (UI foundation)**: shared Modal / EmptyState / Toast /
  FormField primitives, empty states with cross-navigation across the
  app, Risk Register search + pagination.
- **Onboarding**: fresh installs get a quick-start modal with a
  "Load sample data" option that seeds a fully-populated demo
  engagement (SoA, CAPA, audit, review, KPIs, registers).

## Upgrade notes

- Migrations v6–v9 run automatically on first boot; existing data is
  untouched (covered by a dedicated upgrade-path test).
- Encrypted stores migrate identically — new keys participate in
  encryption like every other key.
- Storage version after upgrade: 9.
